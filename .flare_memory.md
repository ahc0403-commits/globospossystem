# Flare Memory

- For POS integration plans, not-yet-implemented Deliberry operational order flows must be classified as Blocked or Decision Needed until implementation and contracts are confirmed. Reserve No-Go for unsafe failures, data contamination, broken coupling, or attempts to claim Full Go while D1 remains blocked.
- Deliberry operational order state machines must never let `DELIBERRY_ORDER_RECEIVED`/`new` downgrade an actioned or terminal order. Stale or out-of-order events belong in anomaly/reconciliation evidence, not state mutation.
- Do not claim POS app pilot execution PASS or 70% user-test PASS from account auth/profile checks alone. A successful app-level user-flow smoke or equivalent command-backed evidence for at least 35/49 pilot cases is required.
- Do not claim POS production deploy readiness unless required POS pilot Auth accounts are checked against production Supabase Auth/profile data, or the Auth check is explicitly skipped and reported as a blocker-risk. Deployment automation must not create, print, or reset pilot passwords.
- POS pilot Auth/profile readiness checks must mirror the app login path: public.users is resolved by public.users.auth_id = auth.users.id. Do not join public.users.id to auth.users.id when judging login readiness.
- Print-routing enqueue batch numbers must be computed once per enqueue event before looping copy types; never recompute after inserting the first destination, or kitchen/floor delta jobs drift into different batch numbers.
- SECURITY DEFINER database functions that set a restricted search_path must schema-qualify extension functions, and contract tests should exercise the callable RPC path rather than only inspecting function text.
- SQL contract tests that depend on auth.uid()-based roles must seed the required auth/users fixture rows inside the transaction instead of relying on local or production seed data.
- POS migrations must be replay-safe: guard table and policy creation against existing objects.
- Legacy `SECURITY DEFINER` RPCs must revoke execution from `PUBLIC`, `anon`, and `authenticated` unless they enforce internal `owner_type` and validate the complete entity-brand-store tuple while preserving required internal compatibility.
- Production migrations must preflight every referenced optional schema, table, and helper; missing dependencies must block before mutation or be explicitly deferred.
- Reassignment operations must preserve immutable ownership and assignment history, including actor, timestamp, source, and destination.
- Production deployment scripts must pin approved project and environment targets; runtime target overrides are forbidden.
- Every production migration must include executable preflight, verification, and rollback assertions for prerequisites, invariants, and restoration behavior.
- Migration backups must remain immutable across replays, and rollback tests must prove exact original-state restoration with all migration-generated history absent.
- Verification SQL must assert every required backup object and use anti-join negative tests so backed-up records missing from the live table cannot be hidden by inner joins.
- Mutation RPCs that create or change legal-entity assignments must atomically maintain append-only active-period history and include executable RPC tests.
- Run production SQL with fail-fast single-transaction semantics (`psql -X -v ON_ERROR_STOP=1 --single-transaction --file`) and emit success only after exit code 0; validate the exact deployment invocation rather than substituting `supabase db query --linked -f` for multi-statement scripts.
- Collector preflight must validate every first-write table and permission, not only aggregate or read paths.
- Parse collector configuration strictly: reject unknown boolean values and misspelled keys instead of silently disabling a store.
- Collector failure reporting must run under an unconditional cleanup path so setup and browser failures cannot bypass escalation.
- Persist and audit an explicit scheduled-slot identity; never infer a delayed run's slot from its actual start timestamp.
- Persist the intended local schedule date and time independently of execution time, with a cross-midnight delayed-run test preserving the intended 22:30 attribution.
- Never continue a migration after a history mismatch or unresolved confirmation warning; stop and verify the applied migration chain first.
- Block production deploys from dirty worktrees or commits not descended from the approved main branch.
